# CLAUDE.md

## Project

**medgemma-studio** — Streamlit application for analyzing medical text and images using Google MedGemma on Apple Silicon with MLX.

## Commands

```bash
uv sync                           # Install dependencies
uv run streamlit run streamlit_app.py  # Run the app
uv run ruff check .               # Lint
uv run ruff format .              # Format
uv run ty check                   # Type check
uv run pytest                     # Run tests
```

When working with Python, invoke the relevant `/astral:<skill>` (`/astral:uv`, `/astral:ty`, `/astral:ruff`) to ensure best practices are followed.

## Claude Code hooks

`.claude/settings.json` (checked in, shared) wires the commands above into hooks so the same gates run automatically:

- **PreToolUse** (`Edit`/`Write`) — blocks writes to `.env`, `.streamlit/secrets.toml`, and `uv.lock` (protects the `HF_TOKEN` and keeps the lockfile package-manager-only).
- **PostToolUse** (`Edit`/`Write`) — on an edited `.py` file, runs `ruff check --fix` + `ruff format` (silent), then `ty check` (surfaces type errors back to Claude to fix).
- **Stop** — runs `uv run pytest` when a turn ends; failures block the stop and are fed back, guarded against an infinite stop→fix loop via `stop_hook_active`.

Personal overrides belong in `.claude/settings.local.json` (gitignored). The shared config is guarded by `TestHooksConfig` (see Tests).

## Architecture

Single-file app (`streamlit_app.py`) with the following structure:

- **Model loading** — `@st.cache_resource` loads `mlx-community/medgemma-1.5-4b-it-bf16` via `mlx_vlm.load()` + `load_config()`. Returns `(model, processor, config)`.
- **Pure helpers** — Extracted for testability (no Streamlit or model):
  - `parse_response(response, is_thinking)` — splits `<unused94>`/`<unused95>` thinking trace from response
  - `build_messages(prompt, system_instruction, images=None, image_labels=None)` — constructs chat message list for `apply_chat_template`, appending one image placeholder per image in `images` (supports any count: text-only, single, comparison pair, or a CT slice stack); when `image_labels` is given, a label text part is interleaved before each image
  - `get_generation_params(has_image, is_thinking, system_instruction, is_localizing=False, is_comparing=False, is_ct=False, is_wsi=False)` — returns `(full_instruction, max_new_tokens)`; comparison uses a 600-token budget, CT and WSI a shared 2000-token budget (2500 with thinking), each keeping the editable instruction
  - `pad_to_square(image)` — top-left pads an image to a square so localization coordinates map back without an offset
  - `parse_boxes(response)` — extracts `[{"box_2d": [y0, x0, y1, x1], "label": ...}]` from a fenced/bare JSON response; drops malformed or non-numeric boxes
  - `scale_box(box_2d, size)` — converts a `[y0, x0, y1, x1]` box normalized to `[0, 1000]` into pixel `(x0, y0, x1, y1)` corners
  - `draw_boxes(image, boxes)` — draws labeled boxes on a copy of a square image
  - `normalize_hu(hu_slice, lo, hi)` — clips a Hounsfield-unit slice to a window and rescales to 0–255
  - `window_ct_slice(hu_slice, windows=CT_WINDOWS)` — packs three fixed HU windows (wide / soft-tissue / brain) into the R/G/B channels of one false-color image; `CT_WINDOWS` is the model's *trained* CT input format, so it is not user-tunable
  - `subsample_indices(n, max_slices)` — uniformly picks slice indices across a volume (endpoints included)
  - `load_ct_volume(dicom_files, max_slices)` — reads per-slice DICOM files, sorts by InstanceNumber, subsamples, and converts to HU via `pydicom`'s `apply_rescale`. Rewinds each upload (`seek(0)`) before `dcmread`, so a repeat Run on the same `session_state`-held `UploadedFile` streams (whose position `dcmread` advances) doesn't fail mid-stream
  - `ram_aware_slice_cap(total_ram_gib=None)` — returns `(default, max)` CT slice / WSI patch counts scaled to installed memory (`(8, 16)` on a 32 GiB Mac; clamped to a hard max of 64); detection in `_detect_total_ram_gib`, memoized for the app by the `@st.cache_data` wrapper `_cached_total_ram_gib` so the slider doesn't re-detect on every rerun (the unit tests call the uncached helper directly or pass `total_ram_gib`)
  - **WSI (whole-slide pathology) helpers** — `mag_from_mpp(mpp_x)` (objective power ≈ `10 / µm-per-pixel`); `effective_magnification(objective_power, downsample)`; `pick_level(level_downsamples, objective_power, target_mag)` (nearest pyramid level to a requested magnification); `patch_grid(level_w, level_h, patch_size)` (non-overlapping 896px tile grid, partial edge tiles dropped, row-major); `tissue_mask(thumbnail_rgb, sat_threshold)` (numpy saturation proxy — `max(RGB)-min(RGB)` — so white/grey glass is excluded); `tissue_patches(grid, mask, level_size, patch_size, min_fraction)` (keep tiles ≥25% tissue, projecting level coords onto the thumbnail-scale mask); `mark_patches(thumbnail, coords, level_size, patch_size)` (draw the sampled patches on the thumbnail, like `draw_boxes`)
  - `load_wsi_patches(uploaded_file, target_mag, max_patches, ...)` — thin OpenSlide loader (touches `openslide`/`tempfile`, like `load_ct_volume` touches `pydicom`): spills the upload to a temp file (OpenSlide opens by path), resolves base objective power (`objective-power` property → `mpp-x` → 40× default), picks the level nearest `target_mag`, tiles → tissue-filters → caps via `subsample_indices` → reads each patch (`read_region` takes a **level-0** location but a target-level size; RGBA out-of-bounds is composited onto white). Returns `(patches, overlay, actual_mag)`; raises `ValueError` for an unreadable slide, a slide too small for one patch, or no detectable tissue; closes the slide and unlinks the temp file in `finally`.
- **UI-layer helpers** (touch Streamlit, not unit-pure): `load_uploaded_image` (open an upload into a PIL image — `image.load()` forces the decode so invalid data fails here, not later at `st.image`; preview is left to the caller so the CXR tab can lay two studies out side by side), `run_model` (format the prompt → **stream** via `mlx_vlm.stream_generate` through `st.write_stream` → return the full accumulated string, error-handling; passes `REPETITION_PENALTY`/`REPETITION_CONTEXT_SIZE` to keep greedy decoding from looping. Streaming turns the long local generation from a blank wait into visibly arriving tokens without violating the single-Q&A constraint — `generate` is itself just `text += chunk.text` over `stream_generate`, so the returned string is identical and `temperature=0` determinism is preserved. **On success each tab stores the returned string and `st.rerun()`s**, so the just-streamed raw output (including the thinking sentinels / raw localization JSON) is discarded and replaced by the clean persisted render — this avoids duplicate output *deterministically*, unlike an earlier `st.empty().empty()` attempt that did not reliably clear the streamed slot inside `st.tabs`+`st.fragment`. On failure the tab does **not** rerun, so the `st.error` stays visible), `render_thought` (split trace into an expander, return the answer), `tab_settings(key_prefix, default_instruction, auto_switch=False)` (per-tab system instruction + thinking toggle with independent session-state keys, tucked inside a collapsed `st.expander("Model settings")` so the primary prompt→upload→Run flow leads each tab; the widgets are still created every run, so auto_switch tracking and key lookups are unaffected).
- **Main UI** — `main()` loads the model once and renders four `st.tabs` (labels carry inline `:material/…:` Material Symbol icons); `st.set_page_config` sets a `page_icon` and `layout="centered"`. Each tab delegates to a `render_*_tab` function with its own keyed widgets. Each `render_*_tab` is decorated `@st.fragment` so a within-tab interaction (slider drag, instruction edit, toggle) reruns only that tab's body rather than re-executing all four tabs; this is sound because each tab is self-contained — it reads the model tuple via its args (which a fragment rerun replays), writes only its own `{ask,cxr,ct,wsi}_result` key, and renders its own `sig`-gated result. The tabs are:
  - **Ask** (`render_ask_tab`) — text-only Q&A, no image; uses `DEFAULT_INSTRUCTION_TEXT`.
  - **Chest X-ray** (`render_cxr_tab`) — single image, two-image comparison, and anatomy localization. Default persona is `DEFAULT_INSTRUCTION_IMAGE` (or `DEFAULT_INSTRUCTION_COMPARE` for two images); `tab_settings(..., auto_switch=True)` tracks that default until the user edits it.
  - **Computed Tomography** (`render_ct_tab`) — DICOM multi-file uploader + a RAM-aware slice-count slider; uses `DEFAULT_INSTRUCTION_CT`.
  - **Pathology (WSI)** (`render_wsi_tab`) — single whole-slide upload (`.svs/.ndpi/.tif/.tiff`) + a magnification `segmented_control` (5/10/20/40×, one-tap objective-power modes; `or WSI_DEFAULT_MAG` guards the single-select None case) + a RAM-aware patch-count slider; uses `DEFAULT_INSTRUCTION_WSI`.
- **Localization** (CXR tab) — When "Locate anatomy" is on (enabled only for exactly one image), the image is padded to a square and a dedicated `LOCALIZATION_INSTRUCTION` (which overrides the system instruction) asks MedGemma for boxes as `[y0, x0, y1, x1]` normalized to `[0, 1000]`. The response is parsed, scaled, drawn, and cropped back to the original size.
- **Comparison** (CXR tab) — When two images are attached, both are sent in one prompt (each labeled "First image:"/"Second image:" so the persona's ordinal wording binds to a specific image) and previewed side by side in `st.columns(2)` (seeing both studies at once is the point of a longitudinal read), the default instruction switches to `DEFAULT_INSTRUCTION_COMPARE`, localization is disabled, and the token budget grows to 600 (1600 with thinking). Mutually exclusive with localization.
- **CT analysis** (CT tab) — Per-slice DICOM files → `load_ct_volume` (HU arrays) → `window_ct_slice` (false-color RGB) → `build_messages` with `"SLICE n"` labels → `run_model` (streams) at the CT token budget. The slice count is capped by `ram_aware_slice_cap` (multi-image inference is memory-heavy on unified memory). A sample windowed slice is previewed. The *preprocessing* (DICOM read + windowing) runs inside an `st.status` that narrates its phases ("Reading DICOM series…" → "Windowing slices…"), then resolves to `state="complete"` (or `state="error"`, `expanded=True` so the nested `st.error` stays visible, on a read failure); **generation then streams in the main area below the status**, so its live tokens aren't buried in a collapsed box. On success the tab `st.rerun()`s (see `run_model`), which sheds the streamed duplicate but also discards the now-transient status; the `state="error"` status persists only because the failure path does *not* rerun.
- **WSI analysis** (Pathology tab) — One whole-slide upload → `load_wsi_patches` (pick magnification level → 896px tissue patches → RGB) → `build_messages` with `"PATCH n"` labels → `run_model` (streams) at the shared 2000-token budget (2500 with thinking). The patch count is capped by `ram_aware_slice_cap` (same memory pressure as CT slices; the OpenSlide reference notebook uses 125 patches, far beyond a local unified-memory budget). A tissue-overview overlay (sampled patches outlined) and a sample patch are previewed, with the **actual** effective magnification disclosed (clamped to the slide's available levels). Mirrors the CT tab's "reduce a high-dimensional input to ≤N 896px tiles → multi-image prompt" pattern; reuses the entire model-side spine, including the same `st.status`-for-preprocessing ("Reading slide and sampling tissue…", then complete/error) followed by streamed generation in the main area.
- **Result persistence** — Each `render_*_tab` runs inference **only** inside the `Run` button block, stores the finished output (answer text / localization annotation+boxes / CT-slice or WSI previews, plus `is_thinking`) in `st.session_state["{ask,cxr,ct,wsi}_result"]`, and renders it *outside* the button gate so it survives later reruns (a button reads `True` only on the run that clicked it; without this the result vanished on the next interaction). Each stored result also carries a `sig` of the run-defining inputs — prompt, upload identity (`_file_sig` = name+size), localize/comparison mode, slice/patch count, magnification — and the render is gated on the live `sig` still matching, so a stale result is **dropped** when an input changes without re-running (swapping an image, toggling localize off, moving a slider), instead of mismatching the visible inputs. The gate is the single shared helper `fresh_result_or_hint(key, live_sig)`: it returns the stored dict when fresh, and when a result exists but is stale it shows an `st.info` hint ("Inputs changed since this result — click Run to refresh.") and returns `None` — so the panel no longer silently vanishes (which read as a bug in a clinical tool). Editing `thinking` (captured in the stored result) is intentionally *not* part of the `sig`, so a result persists across it (no hint).

## Tests

- `tests/test_streamlit_app.py` — pure helpers (`parse_response`, `build_messages`, `get_generation_params`, `pad_to_square`, `parse_boxes`, `scale_box`, `draw_boxes`, `normalize_hu`, `window_ct_slice`, `subsample_indices`, `load_ct_volume`, `ram_aware_slice_cap`, plus the WSI helpers `mag_from_mpp`/`effective_magnification`/`pick_level`/`patch_grid`/`tissue_mask`/`tissue_patches`/`mark_patches`/`load_wsi_patches`); no Streamlit or model needed. `load_ct_volume` is tested with real in-memory DICOMs built by `_dicom_bytes` (so it exercises `dcmread` → `pixel_array` → `apply_rescale`, including out-of-order InstanceNumber sorting). `load_wsi_patches` is tested against a `_FakeSlide` mock (OpenSlide-readable pyramidal slides are fiddly to synthesize, so the pure helpers carry the numeric coverage and the mock exercises the orchestration + the level-0 `read_region` coordinate math). `ram_aware_slice_cap` is tested via its `total_ram_gib` override so it is host-independent. A `TestMlxVlmContract` class introspects the **real** mlx-vlm (static introspection only — no model load) to assert that `stream_generate`/`apply_chat_template`/`load`/`load_config` and `GenerationResult.text` (the `.text` on the chunks `run_model` streams) still match what `run_model`/`load_model` call, including the `repetition_penalty` greedy-loop fix (the sampling kwargs ride on `stream_generate`'s `**kwargs`; their honoring is guarded via `generate`'s public docstring, since `generate` is just `stream_generate` + accumulation); it is the only guard that catches an mlx-vlm upgrade breaking the API, since every other test mocks `mlx_vlm.*`. Imports live inside each contract test so a moved path fails only that check, not collection of the whole suite. A `TestThemeConfig` class likewise guards the real `.streamlit/config.toml` asset (not a mock): it must parse, define both `[theme.light]` and `[theme.dark]` (so the OS/browser auto-switch stays enabled), set the core palette in each mode, and use only theme keys the installed Streamlit registers (a typo fails the test instead of degrading to a silent startup warning). A `TestHooksConfig` class guards the real `.claude/settings.json` Claude Code hooks asset the same way: it must parse as JSON, keep the three configured events wired (`PreToolUse` secret/lockfile guard, `PostToolUse` ruff + ty on edits, `Stop` pytest with a `stop_hook_active` loop guard), and every hook command must be syntactically valid shell (checked by running `sh -n -c <command>` against the real interpreter, so a broken hook — which Claude Code would silently no-op — fails the suite).
- `tests/test_app_ui.py` — UI flow via `streamlit.testing.v1.AppTest`. Patch `mlx_vlm.*` (`load`, `load_config`, `apply_chat_template`, `stream_generate`) at the source, not `streamlit_app`, since AppTest re-execs the script each run; WSI tests likewise patch `openslide.OpenSlide` at the source with a `_FakeSlide` (saturated thumbnail → all tissue). Inference is mocked through the `_patch_stream(monkeypatch, generate_mock)` helper: tests still express the double as the old `generate` contract (`(*a, **k) -> object with .text`) and the helper wraps it as a `stream_generate` that yields that object as one chunk — so `run_model`'s `chunk.text` + `st.write_stream` reproduce the full text, and the `gen_args`/`gen_kwargs` captures still hold (`stream_generate` shares `generate`'s call shape). `test_ask_streams_deltas_and_renders_answer_once` patches `stream_generate` to yield **multiple** deltas and asserts they are concatenated and the answer renders exactly once (the success `st.rerun()` discards the streamed copy — AppTest ends on the post-rerun tree, so this count is deterministic); `test_ask_empty_generation_renders_without_crashing` patches a zero-chunk stream and asserts an empty `### Response`, no crash, no error. **Select widgets by key** (e.g. `at.button(key="cxr_run")`, `at.text_area(key="ct_instruction")`), not by position, since tabs render every widget. Per-tab keys: `ask_*`, `cxr_*` (incl. `cxr_image1`/`cxr_image2`/`cxr_localize`), `ct_*` (incl. `ct_files`/`ct_slices`), `wsi_*` (incl. `wsi_files`/`wsi_mag` — a `segmented_control`, selected via `at.segmented_control(key="wsi_mag")` — /`wsi_patches`). Image tests build in-memory PNGs; CT tests build in-memory DICOMs (`_dicom_bytes`) and must chain multiple `.upload()` calls on one element before a single `.run()` (AppTest re-exec resets a fresh element, so separate `.upload().run()` calls keep only the last). Tests assert per-mode `num_images` / token budgets reach the (mocked) `stream_generate`, the localization/comparison mutual-exclusivity guards, the CT windowed-RGB + 2000-token path, and the WSI patch-RGB + `"PATCH n"` labels + 2000-token path (incl. the no-tissue / unreadable-slide error guards). The CT/WSI **error** tests assert the staged `st.status` (via `at.status[0].state`) resolves to `"error"` when the read fails; the success path is *not* asserted this way because its `st.rerun()` discards the status before AppTest captures the final tree. An autouse `_clear_caches` fixture resets `st.cache_data`/`st.cache_resource` before each test, so the memoized RAM value (varied per test by `_force_ram_gib`) and the cached model don't leak across the session. Result-persistence tests assert a result survives an unrelated rerun (toggling `thinking`) **without** re-invoking inference (call-count guard), and staleness tests assert it is dropped — with the "Inputs changed" `st.info` hint (via `at.info`), not a silent vanish — when a run-defining input changes (prompt edit, second image added, localize toggled off, slice/patch slider moved, magnification changed) without re-clicking Run. Layout/polish coverage: each tab exposes a collapsed `"Model settings"` expander (asserted by label), a two-image CXR comparison lays the studies out in `st.columns` (`at.columns` empty for a single image), and the localization list shows only labels (the raw `box_2d` coords are asserted **absent**).
- Optional manual-test assets live in `samples/` (gitignored except `samples/README.md`): a longitudinal chest-X-ray pair in `samples/cxr/`, a CT DICOM series in `samples/ct/`, and a whole-slide image in `samples/wsi/`. The automated suite does not use them (it builds in-memory fixtures).

## Constraints

- **No multi-turn chat** — single Q&A per interaction
- **Package management** — uv (`pyproject.toml` + `uv.lock`); no `requirements.txt`
- **OpenSlide** — `openslide-python` + `openslide-bin` (the native C lib ships as a pure arm64/universal2 wheel, so no Homebrew is needed). Only single-file WSI formats are accepted; `.mrxs` (multi-file) is excluded
- **HF token** — optional; loaded from `.env` via `python-dotenv`. The MLX model repo is ungated, so a token only helps avoid download rate limits
- **Streamlit API** — use `width="stretch"` (not deprecated `use_container_width`)
- **Theme** — a clinical theme lives in `.streamlit/config.toml`: shared structural settings in `[theme]` plus `[theme.light]`/`[theme.dark]` palettes, so the app follows the user's OS/browser mode (defining both subsections is what enables the switch; a lone `[theme]` would lock to one mode). The light side is the calm clinical default; the dark side mirrors it for reading-room use. All keys are validated against the installed Streamlit version
- **Linting** — ruff with a curated rule set (`E`, `F`, `I`, `UP`, `B`, `SIM`, `C4`); see `[tool.ruff.lint]`
