# Sample images for local testing

These images are **not committed** — the image files are gitignored; only this
README is tracked. They are local fixtures for manually exercising the app's
image features (single-image analysis, **Locate anatomy**, and **longitudinal
comparison**). The automated tests do not use them (they build tiny in-memory
PNGs instead), so this folder is optional.

## The longitudinal chest X-ray pair

A "before"/"after" CXR pair, taken from Google's official MedGemma 1.5
[longitudinal comparison notebook](https://github.com/Google-Health/medgemma/blob/main/notebooks/cxr_longitudinal_comparison_with_hugging_face.ipynb).
Using the same pair makes local results comparable to that reference.

| File | Source / attribution |
|---|---|
| `longitudinal_cxr_before.png` | Real chest X-ray — Stillwaterising, CC0, via Wikimedia Commons |
| `longitudinal_cxr_after.png` | Generated by Gemini and reviewed by an in-house board-certified radiologist |

## Download

```bash
mkdir -p samples
base="https://storage.googleapis.com/hai-cd3-foundations-public-vault-entry/med_gemma/colab_example/cxr"
curl -fsSL -o samples/longitudinal_cxr_before.png "$base/longitudinal_cxr_before.png"
curl -fsSL -o samples/longitudinal_cxr_after.png  "$base/longitudinal_cxr_after.png"
```

## How to use them

1. Run the app: `uv run streamlit run streamlit_app.py`
2. Enter a prompt, e.g. *"Compare these two chest X-rays and describe the changes."*
3. Upload `longitudinal_cxr_before.png` in the first slot, then
   `longitudinal_cxr_after.png` in the second slot that appears.
4. The app enters **comparison mode** automatically (the system instruction
   switches to the comparison persona and "Locate anatomy" is disabled).
5. Run, and look for change-over-time language ("interval", "new", "resolved")
   in the response.

A single image also works for normal analysis and for the **Locate anatomy**
(bounding-box) mode, which is enabled only when exactly one image is attached.
